# 02-areas — Ongoing Responsibilities

Areas are domains of ongoing responsibility with no end date. They don't get completed—they get maintained. Unlike projects (which have a defined outcome and deadline), areas persist indefinitely and represent the recurring commitments that define how you operate.

**Areas generate projects.** When a standard in an area slips, a project gets created to fix it. When an opportunity appears in an area, a project gets created to capture it. Areas are the soil; projects are the crops.

## Areas vs. Projects

| | Areas | Projects |
|---|---|---|
| **End date** | None — ongoing | Yes — defined deadline |
| **Success measure** | Standard maintained | Outcome achieved |
| **Example** | Health | Run a marathon |
| **Example** | Finances | Build 6-month emergency fund |
| **Example** | Brand | Launch Cipher content series |
| **Review cadence** | Weekly check-in | Per milestone |

## Optional Modules

These modules extend your operating system with specialized agents, frameworks, and tracking systems. Each module is self-contained and can be activated independently.

| Module | Folder | What It Adds |
|--------|--------|--------------|
| Health Data Integration | `health-data/` | Wearable data, health briefings, experiment tracking |
| Learning System | `learning/` | Curriculum, study sessions, spaced repetition |
| Relationship Management | `relationships/` | Contact tracking, decay monitoring, meeting prep |
| Financial Organization | `finances/` | Budget awareness, financial reviews, goal tracking |
| Brand & Content | `brand/` | Voice guide, content queue, engagement coaching |
| Career & Role Management | `career/` | 1:1 prep, stakeholder management, career capital |

To activate a module, run `/onboard` and select it.

## How Areas Work in This System

- Each module folder contains its own README (this file), plus data files, frameworks, and agent configuration once activated
- Modules integrate with core commands: `/reflect`, `/plan`, `/review`, `/triage`
- The agent monitors each activated area for standards slipping and surfaces them proactively
- Areas are reviewed weekly — not daily — to avoid micro-managing ongoing work
