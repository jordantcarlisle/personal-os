# Chief of Staff — Triage

Scan all inbound sources and surface what needs attention. Adapt based on time of day.

## Morning Triage (full scan)

Run these checks in parallel where possible:

1. **Calendar:** Check today's events and upcoming 24 hours. Flag:
   - Meetings starting within 2 hours (need prep?)
   - Back-to-back meetings with no buffer
   - Conflicts or double-bookings

2. **Email** (if connected): Scan for:
   - Urgent or time-sensitive items
   - Items from key stakeholders
   - Items requiring a decision today
   - Surface top 3-5 items only — don't overwhelm

3. **Messaging** (if connected): Scan for:
   - Direct mentions and @s
   - Messages in priority channels
   - Action items directed at the user

4. **Inbox:** Read `00-cockpit/inbox.md`. Report unprocessed item count.

5. **Projects:** Scan `01-projects/` for:
   - Projects with no clear next action (GTD violation — flag these)
   - Next actions unchanged for 2+ weeks (stale — flag these)

6. **Waiting For:** Check dashboard Waiting For table:
   - Flag items older than 2 weeks
   - Suggest follow-up actions

7. **Priority Synthesis:** Organize everything into:
   - **Urgent + Important:** Do first
   - **Important, not urgent:** Schedule today if possible
   - **Urgent, not important:** Delegate or batch
   - **Neither:** Drop or defer

## Mid-Day Triage (quick pulse)

1. New urgent items since morning?
2. New inbox captures?
3. Afternoon calendar check
4. "Has anything changed since this morning?"

## Evening Triage (wind-down)

1. Final check for urgent items
2. Process remaining inbox if possible
3. Write tomorrow handoff notes to dashboard Session Context section
